# Remoting

Whether it is needed to Control remote Desktops, or consolidated remote SSH session, there are solutions to help manage multiple Hosts.

## Requirements

- Free and Open source, preferably 0$ cost
- Ability to control Linux, Windows and anything basically

## Contenders

- [Remotely](https://github.com/immense/Remotely) - A remote control and remote scripting solution. Dead since 2024.
- [MeshCentral](https://meshcentral.com/) - Run your own web server to remotely manage and control computers on a local network or anywhere on the internet. Interface is a bit dated.
- [RustDesk](https://rustdesk.com/) - An open-source remote desktop application designed for self-hosting, as an alternative to TeamViewer.
- [HopToDesk](https://www.hoptodesk.com/) - A Direct fork of RustDesk with some minor changes in architecture and protocols. However a beef is ongoing between the two, about competence, licensing and good/ill intentions.
- [HelpWire](https://www.helpwire.app/) - Similar to RustDesk, but also able to better handle Clients and Teams
- [Apache Guacamole](https://guacamole.apache.org/) - A clientless remote desktop gateway. It supports standard protocols like VNC, RDP, and SSH. A bit dated though.
- [Termix](https://docs.termix.site/) - A clientless web-based server management platform with SSH terminal, tunneling, and file editing capabilities.
- [XPipe](https://xpipe.io/) - A connection hub that allows you to access your entire server infrastructure from your local desktop. Supports SSH, Docker, RDP, VNC, Tailscale, Netbird and more.
- [NexTerm](https://github.com/gnmyt/Nexterm?ref=noted.lol) - Server Management Software to connect remotely via SSH, VNC and RDP, deploy docker apps, manage Proxmox LXC and QEMU containers.
